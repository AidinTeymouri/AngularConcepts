Router Resolve

In order to do some task when routing from one component to another we can make use of Router resolve which will only load the page when all the xyz tasks have been done and we can add a loading spinner to it to display it to the user. For eg :- Take the scenario where you want the user to show datatable which has millions of records which has to be fetched via a http call, here if you donot use resolve the component will load but the content will take some time to load which will reduce the user experience so instead we make use of router resolve display the component when data is ready and in the mean while when the resolve is working we can display a loading screen enhancing UX. We can also make use of Resolves for checking authentic users and guide them to appropriate routes same as with guards. but the general use case of resolve is to load a component when some task is done.
